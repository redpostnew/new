**레드포스트 어드민-- \[대행관리\] 핵심기능 설명서**

**요약**\
주문 접수 → QR 발급 → 입고(중국 창고) → 검수·촬영(모바일) → 보관 →
출고(중국→한국) → 통관·국내배송 → CS·정산\
모든 단계에서 QR + DB을 연동해 자동화함 → 오류 감소, 처리 속도 향상,
클레임 대응 용이함.

**0. 준비 단계 (주문 수집 및 QR 준비)**

**1) 주문·검수 항목 수집**

-   고객이 주문할 때 **기본 검수 항목**과 **선택 검수 항목**을 입력하게
    함.\
    예: 색상, 사이즈, 오염 여부, 택 사진 필요 여부 등.
    (네이버폼/카카오폼/자체 주문폼 등 사용)

-   입력 내용은 자동으로 주문 DB(엑셀 출력기능)에 저장됨.

**2) QR코드 생성 및 부착**

-   주문 정보를 DB에 정리한 뒤 QR 이미지를 자동 생성했함\
    생성한 QR을 라벨 프린터로 출력 → **상품 또는 박스에 부착**용도

**3) QR코드에 담을 내용(옵션)**

-   **옵션 A --- 검수 항목 전체 포함**\
    예: A001 \| 가방 \| 봉제, 지퍼, 오염, 포장상태, 택사진

    -   장점: 스캔만으로 바로 모든 검수 항목 확인 가능(인터넷 불필요).

-   **옵션 B --- 주문번호(ID)만 포함**\
    예: A001

    -   장점: QR이 짧아 관리 깔끔.

    -   단점: DB 조회(인터넷 필요)해야 상세 항목 확인 가능.

**1. 입고 단계 (중국 창고 도착 시 처리)**

**1) 입고 확인 절차**

-   창고에서 물건 도착 시 **운송장(트래킹) 바코드**를 스캔함 → 고객 주문
    DB와 단순 비교하여 노데이터(고객 주문에 없는 물건) 판별함.

-   스캐너(핸드스캐너/모바일 앱)로 바코드/QR을 찍으면 자동으로 DB에 입력
    또는 조회되도록 설정함.

**2) QR라벨 발급·부착**

-   입고 확인 후 **주문 DB에서 미리 QR 라벨 발급**함.

-   QR 라벨에 담는 예시: **고객 주문번호, 사서함 번호, 트래킹번호,
    상품사진 링크** 등.

-   입고된 상품에 QR라벨을 붙여 **검수 단계에서 바로 식별** 가능하게 함.

**3) 입고 DB(기록 항목)**

-   반드시 기록할 항목(예시):\
    ORD12345 \| BOX1001 \| BAG-BLACK-M \| TRACK0001 \| 2025-09-10 10:00
    \| 사서함01 \| 노데이터: N

-   주요 컬럼: 트래킹번호, 상품번호, 입고시간, 주문번호,
    고객번호(사서함), 상품명, 노데이터 여부, 노데이터 사진(첨부 링크).

**4) 입고시간**

-   입고 후 보관 기준 예:

    -   0일\~7일: 무료

    -   8일\~30일: 방치 상태(알림 필요)

    -   30일 이후: 유료 보관 시작

-   노데이터(트래킹번호가 주문 DB에 없을 때): 관리자에게 상자 사진
    업로드 프로세스 필요.

**5) 기대 효과 (입고 단계)**

-   박스 개봉 전에도 어떤 고객 물건인지 확인 가능함.

-   여러 고객 물품 혼재시에도 스캔 한 번으로 분류 가능함.

**2. 검수·촬영 단계 (상품 상태 확인 및 부가서비스)**

**1) 검수 시작(모바일)**

-   검수자가 상품 박스 QR을 **스캔**하면 검수 화면이 자동으로 열림.

-   화면에서 **사진 촬영, 수량 체크, 상태 기록**을 바로 입력함.

-   검수 입력은 모바일에서 체크박스/사진 업로드 형태로 간편하게
    수행되며, 저장 시 자동으로 **검수 상태 = 완료**로 변경됨.

**2) QR → 검수 체크리스트 연결 방식**

-   **방법 A**: QR에 검수 항목을 텍스트로 포함 → 스캔 시 바로
    표시(오프라인 가능).

-   **방법 B**: QR에 주문 ID만 포함 → 스캔 시 DB URL 호출해 검수 항목
    불러오기(온라인 필요).

-   DB 등에서 레코드 열기(웹UI 구현) → 직원이 체크리스트 필드에 체크 후
    사진 첨부.

**3) 검수 체크리스트(예시 항목)**

-   **기본검수 항목**

    -   상태: 정상/오염/파손/오배송

    -   수량: 일치/불일치

    -   사진첨부: 사진 파일 경로 또는 클라우드 링크

    -   검수일시: 자동 기록

-   **검수 기록 예 (엑셀 행)**:\
    ORD12345 \| BOX1001 \| 가죽가방 \| BLACK \| 1 \| 정상 \|
    /검수사진/ORD12345.jpg \| 2025-09-10 14:25

**4) 검수 코드(구분 ID)**

-   검수 체크리스트 종류를 구분하는 ID 부여함.

    -   예: CHK_BAG01 = 가방류 검수 체크리스트

    -   예: CHK_TOY02 = 장난감 검수 체크리스트

-   검수코드를 통해 자동으로 해당 체크리스트를 불러오고, 검수자가 혼동
    없이 작업함.

-   검수항목은 7개 분류 63개 항목임

**5) 사진·증빙 관리**

-   검수 사진은 파일 경로/클라우드 링크로 저장되어 고객·관리자가
    언제든지 확인 가능함.

**6) 비용·서비스 구분**

-   **구매대행(업체 소유 상품)**: 검수 기본 무료로 제공 가능함.

-   **배송대행(고객 소유 상품)**: 검수는 유료 서비스로 설정 가능함.

**7) 검수의 효과**

-   디지털 검수 기록으로 클레임 대응이 쉬워짐.

-   누락·착오가 감소함.

-   고객별 검수 리포트 자동 생성되어 투명성 증가.

**3. 보관 단계 (창고 → 출고 전)**

**1) 보관 상태(예시)**

-   **출고대기**: 고객 결제 대기 또는 출고 지시 대기 상태.

-   **오류입고**: 입고 오류 발생 → 오류무시(고객이 그냥
    받음)/출고보류/반품관리 처리 가능.

-   **재고신청**: 고객 요청 시 장기 보관(재고신청 → 재고입고 → 재고등록)
    처리함.

**2) 보관 단계의 효과**

-   위치 파악 속도 증가 → 재고 찾는 시간 단축.

-   분실·누락 감소.

**4. 출고 단계 (중국 → 국내 발송)**

**1) 출고 준비(라벨/서류)**

-   **출고 전 라벨에 QR 인쇄** → 스캔하면 고객 주문정보, 통관정보,
    운송장번호 등 확인 가능함.

-   출고 서류 준비: 상업송장(Commercial Invoice), 패킹리스트(Packing
    List), 인보이스, B/L(Air/Sea), 통관서류 등.

**2) 출고 진행 순서 (세부)**

1.  **QR 스캔** → 주문정보 및 검수/촬영 이력 확인함.

2.  **나눔/묶음 관리**: 합배번호·나눔번호·송장번호 등 필드로 묶음관리.

3.  **포장요청**: 리패킹, 라벨링 등 추가작업 지시.

4.  **포장DB 기록**: 출고정보, 추가작업, 패킹관리(중국→한국) 기록함.

5.  **출고검수**: 포장확인 및 출고검수 진행.

6.  **집하 및 선적**: 창고→항구/공항으로 집하, B/L

7.  **운송 & 추적**: 운송사 인계 후 스캔/인수증 처리되면 출고완료 처리.

    1.  해운: B/L, 운송장번호=B/L로 대체, Yida API 연결

    2.  항공: CJ대한통운, API 연결

    3.  택배: CJ대한통운, API 연결

**3) 출고 상태 관리(예시 흐름)**

-   출고대기 → 고객 결제대기/완료 → 포장완료 → 출고검수 완료 → 운송사
    인계 완료 → 출고완료 → 배송중 → 배송완료

**4) 출고 단계의 QR 활용 효과**

-   직원이 QR만 찍어도 출고 내역이 자동 업데이트됨.

-   고객이 QR을 스캔하면 운송장 추적 페이지로 연결되어 CS 문의 감소함.

**5. 국내배송 단계 (통관 → 국내 배송 → 고객수령)**

**1) 통관 및 관부가세**

-   관부가세 결제 및 수입신고 진행 후 통관 완료함.

**2) 배송 방식 구분**

-   **배송대행 (고객 소유 상품)**: 업체가 수출신고 대행, 고객에게 배송
    추적 제공.

-   **구매대행 (업체 소유 상품)**: 업체가 자체 수입신고 및 B/L 발행,
    결제/운송 일괄 관리.

**3) 고객용 QR 활용**

-   최종 패키지에 QR 부착 → 고객이 스캔하면 **배송조회/고객센터/리뷰
    작성/재구매 쿠폰** 등으로 연결됨.

-   QR을 통해 마케팅(쿠폰, 이벤트)과 고객지원 기능을 동시에 제공함.

-   고객이 박스 QR을 찍으면 자동으로 **주문정보가 불러와져**
    고객센터에서 즉시 확인 가능함.

**4) 효과**

-   고객 스스로 배송조회가 가능해 CS 문의가 줄었음.

-   리뷰·재구매 전환율 상승.

**6. 정산 (수수료·보관료·출고요금 등)**

**1) 정산 DB 구성(예시)**

-   **기본**: 배송비 요율표, CBM(부피) 요율표

-   **부가서비스**: 라벨링, 검수유료, 리패킹, 사진/보고서 서비스 등

-   **기타**: 반품 수수료, 부피무게 적용률, 카드수수료

-   **결제DB**: 결제금액, 결제방법, 주문정보, 상품정보, 회원정보 연동

**2) 자동 결제·청구**

-   예치금 자동 결제 기능 구현 가능(성공/실패 기록 유지).

-   예치금 KRW, CNY 등 통화 구현

-   출고 이후 청구 항목 및 3PL 입고 후 청구 항목을 자동 정산하도록
    설정함.

-   BL번호 등 서류는 엑셀 업로드로 일괄 등록 가능.

**7. 업무흐름(간단 요약)**

1.  고객 주문 접수 → 검수 항목 입력 → 주문DB 저장

2.  DB나 엑셀에서 QR 자동 생성 → 라벨 출력 → 상품에 부착

3.  창고 도착 → 트래킹 스캔 → 입고DB 기록 → QR 부착

4.  검수자 QR 스캔 → 사진·수량·상태 입력 → 검수DB 저장 → 검수 완료 표시

5.  랙 위치 QR로 보관 위치 매핑(예정)

6.  출고 요청 시 QR 스캔 → 검수·포장 확인 → 서류 준비 → 선적(B/L/) →
    출고완료

7.  한국 도착 → 통관 → 국내 배송 → 고객 수령→ CS (최종 QR로 리뷰/쿠폰
    유도)
