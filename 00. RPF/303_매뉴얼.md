## 1. 화면 개요

- **페이지명** — 결제관리
- **목적** — 결제번호 조회 · 입금 확인 · 결제 취소 처리
- **주요 수행 업무**
  - 결제번호별 결제 현황 조회
  - 무통장 입금 수동 확인 처리
  - 결제 취소 처리 및 자동 상태 전환 확인
  - 결제내역 상세 및 주문서 조회

## 2. 결제 업무 흐름

### 2.1 정상 결제 흐름

```
청구완료 → 결제대기
    ↓
┌─────────────────────────────────────────┐
│  예치금·카드   │  무통장입금             │
│  자동 결제     │  입금 후 관리자 수동    │
│  처리          │  입금확인 처리          │
└─────────────────────────────────────────┘
    ↓
결제완료
    ↓
구매완료 / 출고완료 자동 전환
```

### 2.2 결제 취소 흐름

```
결제완료
    ↓
결제 취소 처리
    ↓
결제취소 상태 → 결제대기 자동 전환
```

> ⚠️ 일부 상태에서는 취소가 불가할 수 있음

### 2.3 결제 상태 코드

| 상태 | 설명 |
|------|------|
| 미입금 | 입금이 아직 확인되지 않은 상태 |
| 미확인 | 입금됐으나 관리자 확인 전 상태 |
| 결제완료 | 입금 확인 완료 및 결제 처리 완료 |
| 취소됨 | 결제가 취소된 상태 |

### 2.4 결제 방법

| 방법 | 설명 |
|------|------|
| 관리자 예치금결제 | 관리자가 고객 예치금으로 직접 결제 처리 |
| 예치금 결제 | 고객이 보유 예치금으로 결제 |
| 무통장입금 | 고객 무통장 입금 후 관리자 확인 |
| 카드 | 카드 결제 (PG 연동) |

## 6. 입금 확인 처리

### 6.1 개요

- 무통장 입금 내역을 수동으로 확인하는 기능
- 자동 API 확인이 되지 않은 경우 관리자가 직접 처리

### 6.2 처리 흐름

```
무통장 입금 → 미확인 상태
    ↓
관리자 입금확인 처리
    ↓
결제완료 자동 전환
    ↓
주문서 결제완료 시 구매·출고완료 자동 전환
```

### 6.3 입금 확인 모달 구성

| 항목 | 설명 |
|------|------|
| 처리 방식 | 수동 입금확인 |
| 입금확인 후 처리 흐름 | 결제완료로 변경됨 안내 |
| [입금확인 처리] | 확인 처리 실행 버튼 |
| [취소] | 모달 닫기 |

### 6.4 처리 절차

1. 미확인·미입금 상태의 결제 행에서 [입금확인 처리] 클릭
2. 입금 확인 모달 열림
3. 처리 흐름 확인
4. [입금확인 처리] 클릭 → 결제완료 자동 전환
5. 연결된 주문서 → 구매완료 / 출고완료 자동 처리

## 7. 결제 취소

### 7.1 개요

- 결제 완료된 건을 취소 처리
- 취소 후 자동으로 이전 상태로 복구

### 7.2 취소 처리 흐름

```
결제완료
    ↓
결제 취소 처리
    ↓
결제취소 상태 → 결제대기 자동 전환
```

> ⚠️ 일부 상태에서는 취소가 불가할 수 있습니다. 취소 불가 시 안내 메시지 표시

### 7.3 결제 취소 모달 구성

| 항목 | 설명 |
|------|------|
| 해당 결제건을 취소 처리합니다 | 취소 안내 메시지 |
| 취소 후 상태 | 복구될 상태 안내 |
| 결제취소 후 자동 처리 흐름 | 자동 전환 상태 안내 |
| [결제취소] | 취소 확정 버튼 |
| [취소] | 모달 닫기 버튼 |

### 7.4 취소 절차

1. 결제완료 상태의 결제 행에서 [결제 취소] 클릭
2. 결제 취소 모달 열림
3. 취소 후 상태 및 자동 처리 흐름 확인
4. [결제취소] 클릭 → 취소 처리 완료
5. 주문 상태 → 결제대기 자동 전환

> ⚠️ 결제 취소는 되돌릴 수 없으므로 반드시 확인 후 진행
