## 2. 인포그래픽 패널 (상단 현황)

화면 최상단에 서비스 유형별·상태별 주문 건수를 표시한다.

### 2.1 패널 구성

| 컬럼 | 표시 상태 |
|------|----------|
| 구매대행 | 구매견적 / 결제대기 / 결제완료 / 구매완료 |
| 수입대행 | 수입견적 / 결제대기 / 결제완료 / 구매완료 |
| 배송대행 | 접수대기 / 접수신청 |

### 2.2 상태 카운트 클릭

- 카운트 버튼 클릭 → 해당 유형·상태로 목록 자동 필터링
- 예: `구매대행 > 구매견적 [3]` 클릭 → 구매대행·구매견적 3건만 표시

### 2.3 접기 · 펼치기

- 우측 상단 [접기] / [펼치기] 버튼으로 패널 토글
- 접힌 상태에서 헤더에 `구매대행 N건 | 수입대행 N건 | 배송대행 N건` 요약 표시

### 2.4 상태별 툴팁 (`?` 아이콘)

| 상태 | 설명 |
|------|------|
| 구매견적 | 상품가·현지배송비·수수료 합산 후 결제 요청 금액 생성 → 결제대기 전환 + 카톡 알림 |
| 결제대기 | 입금 확인 전 대기 → 1주일 이상 미결제 시 신청취소 자동 이동 |
| 결제완료 | API 연동으로 입금 확인 시 자동 전환 |
| 구매완료 | 관리자 현지 구매 후 직접 변경 → 사이트 금액 < 1688 금액이면 오류 알림 |
| 수입견적 | 상품가·배송비·수수료·물류비·관부과세 포함 한국도착가 기준 결제 요청 금액 생성 |
| 접수대기 | 배송대행 주문서 작성 전 단계 |
| 접수신청 | 배송대행 신청서 작성 완료 단계 |

## 3. 주문 상태 흐름 및 자동화

### 3.1 구매대행

```
구매견적 → 결제대기 → 결제완료 → 구매완료
         → 입고완료 → 부분입고 → 검수완료 → 포장완료 → 출고완료
```

- **구매견적** — 담당자가 견적 모달에서 상품가·환율·수수료 입력 후 [청구하기]
  - → 결제대기 자동 전환 + 고객 카톡 알림 발송
- **결제대기** — 고객 입금 전 유지
  - 1주일 이상 미결제 → 신청취소 자동 이동
  - [입금요청] 버튼으로 수동 독촉 가능
- **결제완료** — API 연동으로 입금 확인 시 자동 전환, 또는 수동 일괄 변경
- **구매완료** — 관리자가 현지 구매 완료 후 직접 변경
  - 사이트 표시 금액 < 1688 금액 → 오류 알림 발생, 반드시 확인 후 변경

### 3.2 수입대행

```
수입견적 → 결제대기 → 결제완료 → 구매완료
```

- 구매대행과 동일한 흐름이나 견적 산출 범위 다름
- 견적 포함 항목 — 상품가 + 현지배송비 + 수수료 + 물류비 + 관부과세 (한국도착가 기준)
- 부가서비스도 견적 단계에서 사전 협의 포함 가능
- 견적 완료 → 결제대기 자동 전환 + 카톡 알림 발송

### 3.3 배송대행

```
접수대기 → 접수신청
```

- **접수대기** — 고객이 주문서 미작성 상태
- **접수신청** — 고객 신청서 작성 완료 → 담당자가 내용 확인 후 이후 단계 처리

### 3.4 자동화 트리거 정리

| 트리거 | 자동 동작 |
|--------|----------|
| 구매견적·수입견적 청구 완료 | 결제대기 전환 + 카톡 알림 발송 |
| API 입금 확인 | 결제완료 자동 전환 |
| 1주일 이상 미결제 | 신청취소 자동 이동 |
| 예치금 자동결제 설정 | 결제 단계 자동 처리 |

## 4. 검색 및 필터

### 4.1 패널 열기·닫기

- [상세 검색] 헤더 클릭 → 검색 폼 펼침 / 접힘 (기본: 닫힘)

### 4.2 기간 및 주문구분

| 필드 | 설명 |
|------|------|
| 주문일자 | 시작일 ~ 종료일 직접 입력 |
| 빠른 선택 | [오늘] [1주] [1개월] 버튼 (기본: 1개월) |
| 주문구분 | 전체 / 구매대행 / 수입대행 / 배송대행 라디오 |

### 4.3 번호·코드 계열

| 필드 | 입력 예시 |
|------|----------|
| 주문번호 | 260311-B001-0001 |

| 상품코드 | prd_1136 |
| 사서함번호 | NRP9 |
| 중국송장번호 | 중국 내 배송 운송장번호 |

| 필드 | 설명 |

| 수취인 | 배송 수취인 이름 |

## 7. 구매견적 작성

### 7.1 모달 열기

1. 대상 주문 [추가기능 ▾] 클릭
2. 드롭다운에서 [구매견적] 선택
3. 구매견적 모달 팝업 열림

### 7.2 원가 정보 입력

| 항목 | 설명 | 방식 |
|------|------|------|
| 구매금액 (CNY) | 현지 상품 금액 | 자동 표시 |
| 현지 배송비 (CNY) | 중국 내 배송비 | 자동 표시 |
| 위안환율 (원/CNY) | CNY → KRW 환율 | 직접 입력 |
| 달러환율 (원/USD) | USD → KRW 참고 환율 | 직접 입력 |
| 실송금환율 | 정산에 필요한 환율 | 직접 입력 |

> 📌 당일 기준 환율 입력 → 아래 계산 항목 자동 업데이트

### 7.3 수수료 및 세금

| 항목 | 설명 | 비고 |
|------|------|------|
| 회원구매 수수료 | 구매대행 수수료율 | 기본 10%, 직접 수정 가능 |
| 구매금액 (원) | 환율·수수료 적용 후 금액 | 자동 계산 |
| 부가세 | 구매금액의 약 10% | 자동 계산 |
| 대행수수료 | 구매수수료의 10% | 자동 계산 |

**계산 공식**
```
구매금액(원) = 구매금액(CNY) × 위안환율 × (1 + 수수료율/100)
대행수수료   = 구매금액(원) × 0.1
부가세       = 구매금액(원) × 0.1
총 청구금액  = 구매금액(원) + 부가세
```

### 7.4 청구 완료

1. 계산 결과 확인
2. [청구하기] 클릭
3. 구매견적 버튼 → ✓ 청구완료 배지로 변경
4. 주문 상태 → 결제대기 자동 전환
5. 고객 카톡 알림 자동 발송

### 7.5 견적취소

- [추가기능 ▾] → [견적취소] 선택
- 발행된 견적을 취소하고 이전 상태로 복원

| 통관 수수료 | 통관 처리 비용 | 자동 계산 |

| 실제 마진율 | 금액 계산 후 실제 마진율 | 자동 계산 |

- [청구하기] 클릭 → 결제대기 전환 + 카톡 알림

## 9. 1688 연동 기능

### 9.1 1688 가격동기화

- **목적** — 현지 1688 최신 상품 가격을 시스템에 업데이트
- **절차**
  1. 동기화 대상 주문 체크박스 선택
  2. [추가기능 ▾] → [1688 가격동기화] 클릭
  3. 선택 주문 상품 가격 → 1688 현재 가격으로 업데이트
- **주의** — 사이트 표시 금액 < 1688 금액 → 오류 알림 → 수동 확인 후 견적 재작성

### 9.2 1688 주문서생성

- **목적** — 선택 주문에 대한 1688 주문서 자동 생성
- **절차**
  1. 대상 주문 체크박스 선택
  2. [추가기능 ▾] → [1688 주문서생성] 클릭
  3. 1688 API 통해 주문서 자동 생성
- 📌 실행 전 상품코드·옵션 정보 정확히 입력되어 있는지 확인

### 9.3 1688 송장동기화

- **목적** — 1688 발송 상품의 중국 송장번호 자동 수신
- **절차**
  1. 대상 주문 체크박스 선택
  2. [추가기능 ▾] → [1688 송장동기화] 클릭
  3. 1688 주문번호 연결 중국 송장번호 자동 입력
- 📌 1688 주문번호가 정확히 입력되어 있어야 정상 동작

## 10. 메모 관리

### 10.1 관리자 메모

- **성격** — 내부 운영용, 고객 비노출
- **작성 순서**
  1. 상품 상세 테이블 [관리자 메모 > 등록/보기] 클릭
  2. 메모 모달에서 내용 입력
  3. [저장] 클릭
- **이력 관리** — 저장 시마다 `YYYY-MM-DD HH:MM` + 내용 자동 누적, 이력 삭제 불가
- **입력 예시**
  ```
  2026-03-11 고객 요청으로 포장 변경. 추가 확인 필요.
  고객 연락처: 010-XXXX-XXXX
  ```

### 10.2 물류센터 요청사항

- **성격** — 물류센터 전달용 특별 요청
- **작성 순서**
  1. [물류센터 요청사항 > 상세보기] 클릭
  2. 요청 내용 입력
  3. [저장] 클릭
- ⚠️ 이력 관리 없음 — 최신 내용만 저장
- **입력 예시**
  ```
  박스 포장 필수 / 에어캡 2중 포장 후 발송
  스티커 부착 위치: 상단 우측 / 온도 민감 상품 주의
  ```

### 10.3 상품 메모

- 상품 행 [상품메모 > 등록/보기] 클릭 → 모달에서 내용 입력·저장

## 13. 나눔(Split) · 묶음(Merge)

### 13.1 나눔 처리

- **목적** — 상품을 사이즈·종류별로 나눠 배송
- **활성화** — 체크박스 2건 이상 선택 시 [나눔] 버튼 활성 (주황색)
- **절차**
  1. 대상 주문 체크박스 선택 (2건 이상)
  2. [나눔] 클릭
  3. 현재 주문정보 확인
  4. 나눌 제품 목록에서 상품·수량 설정
  5. [나눔 확정] 클릭
- **나눔 처리 내역** — 나눔수량 / 잔여수량 표시, 바코드 출력 가능
- ⚠️ 처리 후 새 주문번호 생성, 되돌리기 어려움

### 13.2 묶음 처리

- **목적** — 동일 받는사람 주문 합포장
- **활성화** — 체크박스 2건 이상 선택 시 [묶음] 버튼 활성 (초록색)
- **절차**
  1. 대상 주문 체크박스 선택 (2건 이상)
  2. [묶음] 클릭
  3. 기준 주문 확인
  4. 묶음 대상 주문 목록 확인 (이미지·상품명·상품옵션)
  5. [묶음 확정] 클릭
- ⚠️ 처리 후 개별 주문 합산, 결제금액·상품 구성 확인 필수
