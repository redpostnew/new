## 1. 화면 개요

- **페이지명** — 청구 관리
- **목적** — 청구번호별 주문 조회 및 청구·취소 처리
- **301 청구관리와의 차이**
  - 301: 출고 전 청구번호 생성 및 청구 처리 (주문 중심)
  - 302: 생성된 청구번호별로 주문을 조회하고 청구·취소 처리 (청구번호 중심)
- **주요 수행 업무**
  - 청구번호별 주문 현황 조회
  - 청구하기 처리 (추가 비용 입력 포함)
  - 청구 취소 및 청구번호 일괄 취소

## 2. 청구 업무 흐름

### 2.1 청구 상태 흐름

```
미생성 (청구번호 없음)
    ↓
청구번호 생성 (301 청구관리에서 처리)
    ↓
청구 진행 (결제 요청 발송)
    ↓
청구완료
```

### 2.2 취소 흐름

```
청구완료
    ↓
청구 취소 → 취소됨
        ↓
    청구번호 취소 → 청구번호 무효화
```

### 2.3 주문서 상태별 청구 현황

| 주문서 상태 | 설명 |
|------------|------|
| 포장완료 | 패킹 완료, 청구 대기 |
| 쿠팡서류대기 | 쿠팡 서류 출력 대기, 청구 대기 |
| 결제대기 | 청구 후 결제 대기 중 |
| 구매견적/결제대기 | 구매견적 포함 결제 대기 |
| 신청/결제대기 | 신청 후 결제 대기 중 |
| 반품신청/결제대기 | 반품 신청 후 결제 대기 중 |
| 미생성 | 청구번호 미생성 상태 |

## 6. 청구하기 처리

### 6.1 개요

- 선택한 주문건을 청구 처리 (결제 요청 발송)
- 추가 비용을 직접 입력하여 합산 청구 가능

### 6.2 청구하기 모달 구성

**청구 상태 확인**

| 항목 | 설명 |
|------|------|
| 청구 상태 | 현재 청구 진행 상태 표시 |
| 청구 진행 | 청구 처리 실행 버튼 |

**추가비용 직접 입력**

- 청구서에 추가비용을 직접 입력하여 합산 가능
- 항목명 · 단가 · 단위 · 소계 입력
- [저장] 클릭 → 청구 금액에 반영

### 6.3 청구 절차

1. 청구할 주문 행의 [청구하기] 버튼 클릭
2. 청구하기 모달 열림
3. 추가비용 입력 필요 시 항목 입력 → [저장]
4. [청구하기] 버튼 클릭 → 선택한 주문건 청구 처리
5. 고객에게 결제 요청 알림 발송
6. 주문 상태 → **청구완료** 전환

## 7. 청구 취소

### 7.1 개요

- 개별 주문의 청구를 취소 처리
- 취소 후 상태는 이전 주문 상태로 복구

### 7.2 취소 후 상태

| 취소 전 상태 | 취소 후 상태 |
|------------|------------|
| 청구완료 | 포장완료 / 쿠팡서류대기 복구 |
| 결제대기 | 포장완료 / 쿠팡서류대기 복구 |

### 7.3 청구 취소 모달

| 항목 | 설명 |
|------|------|
| 해당 주문건의 청구를 취소합니다 | 취소 안내 메시지 |
| 취소 후 상태 | 복구될 상태 안내 |
| [청구 취소] | 취소 확정 버튼 |
| [취소] | 모달 닫기 버튼 |

### 7.4 취소 절차

1. 취소할 주문 행의 [청구 취소] 버튼 클릭
2. 청구 취소 모달 확인
3. 취소 후 상태 확인
4. [청구 취소] 클릭 → 취소 처리 완료

> ⚠️ 청구 취소 후 상태는 `취소됨`으로 변경되며, 청구완료 이전 상태로 복구됩니다.

## 8. 청구번호 취소 (일괄)

### 8.1 개요

- 선택한 건의 청구번호를 일괄로 무효화 처리
- 개별 취소와 달리 여러 건을 한 번에 처리 가능

### 8.2 청구번호 취소 모달

| 항목 | 설명 |
|------|------|
| 선택 건수 | 취소할 청구번호 수량 |
| 청구번호 무효화 | 선택 건수 청구번호 무효화 안내 |
| [청구번호 취소] | 취소 확정 버튼 |

### 8.3 일괄 취소 절차

1. 취소할 주문 체크박스 선택 (복수 선택 가능)
2. [청구번호 취소] 버튼 클릭
3. 청구번호 취소 모달에서 선택 건수 확인
4. [청구번호 취소] 클릭 → 선택된 청구번호 전체 무효화

> ⚠️ 청구번호 무효화 후에는 해당 번호로 재청구가 불가합니다. 반드시 건수를 확인 후 진행하세요.
