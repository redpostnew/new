## 지표 산출 조건 
### 1. 👤 회원 및 주문 현황
| 상세 지표 | 데이터 수집 및 산출 로직 (Logic) |
| :--- | :--- |
| **가입 현황 (어제/오늘)** | 회원 테이블에서 '가입 일시'가 각각 어제와 오늘인 데이터의 개수를 카운트합니다. |
| **등급별 회원수** | 전체 회원을 현재 '회원 등급(Level)' 기준으로 그룹화하여 각 등급별 인원수를 집계합니다. |
| **주문 접수 (어제/오늘)** | 전체 주문 내역 중 '주문 생성 일시'가 해당 날짜인 건수를 합산합니다. (취소 주문 제외) |
| **유형별 주문 (오늘)** | 오늘 접수된 주문을 '주문 유형(배송/구매/수입대행)' 코드로 분류하여 각각의 건수를 셉니다. |

### 2. 📦 입고 및 검수 관리
| 상세 지표 | 데이터 수집 및 산출 로직 (Logic) |
| :--- | :--- |
| **입고량 (어제/오늘)** | 물류센터에 도착하여 시스템상 '입고 확인' 처리가 완료된 시점을 기준으로 수량을 집계합니다. |
| **노데이터 상품** | 입고는 되었으나 고객의 주문서(신청서)와 매칭되지 않아 '주인 미상'으로 분류된 상품 수입니다. |
| **기본검수 현황** | 오늘 날짜로 검수 프로세스에 진입한 건 중 '대기', '완료', '오류(파손/오배송)' 상태 건수를 각각 셉니다. |
| **부가서비스 작업량** | 오늘 작업이 완료된 부가서비스를 종류별(라벨링, 포장보완, 사진촬영 등)로 카운트합니다. |

### 3. 🏭 보관 및 출고 관리
| 상세 지표 | 데이터 수집 및 산출 로직 (Logic) |
| :--- | :--- |
| **재고 보관 현황** | 현재 로케이션(위치)이 할당되어 있고 출고되지 않은 '보관 중' 재고를 카테고리별로 합산합니다. |
| **상품 상태별 현황** | 전체 재고 중 현재 상태가 '출고 대기중', '오류 입고', '정상 재고등록'인 건을 각각 필터링합니다. |
| **나눔/묶음 배송** | 오늘 출고 확정된 건 중 '1주문 N운송장(나눔)'과 'N주문 1운송장(묶음)' 케이스를 구분하여 셉니다. |
| **물류 방식별 출고** | 오늘 포장/출고된 건들의 운송 방법(항공특송, 해운택배, LCL 등) 코드를 기준으로 분포를 봅니다. |

### 4. 💰 예치금 및 정산 관리
| 상세 지표 | 데이터 수집 및 산출 로직 (Logic) |
| :--- | :--- |
| **예치금 총액** | 전체 회원의 예치금 지갑 잔액을 통화 단위(KRW, CNY)별로 모두 더하여 산출합니다. |
| **자동결제 현황** | 오늘 실행된 자동 결제 로그 중 결과값이 '성공'인 건과 '실패(잔액부족 등)'인 건을 집계합니다. |
| **정산 예정액** | 오늘 '선적(출고완료)' 처리된 건들에 대해 고객에게 최종 청구된 배송비 및 수수료의 총합입니다. |
| **미결제/부족** | 현재 출고 대기 중인 건들 중, (고객 예치금 잔액 < 결제 청구 금액) 조건에 해당하는 건수와 금액입니다. |
| **선지급 미회수금** | 구매대행 건 중 회사가 비용을 선지출했으나, 고객으로부터 아직 2차 결제(입금)를 받지 못한 금액입니다. |

### 5. 📊 수익 및 비용 분석
| 상세 지표 | 데이터 수집 및 산출 로직 (Logic) |
| :--- | :--- |
| **구매대행 수익성** | (고객 총 청구액) - (실제 상품 매입가 + 현지 배송비)를 계산하여 수수료 수익을 산출합니다. |
| **배송대행 수익성** | (고객 청구 운송비 - 운송사 원가) + (부가서비스 매출)을 합산하여 수익을 계산합니다. |
| **수입대행 마진** | (최초 고객 견적가) - (실제 발생한 모든 매입/통관 비용)을 계산하여 확정 마진을 봅니다. |
| **결제 타입별** | 오늘 발생한 결제 내역을 수단(무통장, 예치금, 카드, 관리자처리)별로 나누어 비율을 산출합니다. |
| **결제 구분별** | 결제 명목이 '배송비 결제'인지 '상품 구매비용 결제'인지 구분 코드로 나누어 집계합니다. |
