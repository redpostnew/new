# HS_CODE

## 배송대행  HS_CODE 활용
- 배대지는 고객이 직접 구매한 상품을 대신 수입 신고하는 구조
-  여기서  HS_CODE는 “통관 자동신고용 상품분류” 로 쓰임

### 실제 시스템 예시
  1. 예: 몰테일, 오마이집, 아이포터 등
  2. 사용자는 상품명만 입력
  3. 시스템이 내부 HS 코드 매핑 테이블에서 가장 근접한 코드 추천
  4.  HS_CODE + 세율 + 품명이 자동으로 신고서에 삽입
  5. 관세사 연계 API로 UNI-PASS 신고 자동화

## 구매대행(쇼핑형 직구)/수입대행  HS_CODE 활용
   - 구매대행은 대행업체가 직접 수입자(통관주체) 로 등록
   -  HS_CODE는 “세금 계산 + 통관 기준”을 모두 좌우하는 법적 키(key)

| 구분                       | 설명                                                 |
| ------------------------ | -------------------------------------------------- |
| **상품 DB에  HS_CODE 필드 포함** | 상품 등록 시  HS_CODE를 필수값으로 저장 (상품 → 코드 1:1 매핑)         |
| **세율 자동 반영**             |  HS_CODE 기준으로 “예상 관세 + 부가세”를 실시간으로 계산 후 고객에게 표시     |
| **정확한 통관 서류 생성**         | 수입신고서 작성 시  HS_CODE + 품명 + 과세가격 자동 전송               |
| **데이터 통합 분석**            | 품목별  HS_CODE별 수입금액/세율/품목 리스크 분석 (특히 전자제품, 의류, 식품군)  |

### 실제 시스템 예시
예: 큐텐(Qoo10), 위즈위드, 스토어팜 구매대행 셀러, 네이버 해외직구센터
  1. 각 상품은 “ HS_CODE” 필드가 있고,
  2. 관리자나 관세사가 최초 등록 시 코드 지정
  3. 이후 SKU별로 자동 세율 계산 → 주문당 세금 산정
  4. 시스템이 “품목별 예상세금”을 결제 페이지에 표시

## 배송대행 vs 구매대행 비교 요약

| 구분                | 배송대행                | 구매대행       |
| ----------------- | ------------------- | --------------------- |
| **통관 주체**         | 개인(고객) + 배대지 위임     | 구매대행 업체 (사업자)         |
| ** HS_CODE 입력 시점** | 고객 입력 or 시스템 자동 추천  | 상품 등록 시 고정 매핑         |
| **주요 목적**         | 수입신고 자동화 / 분류 정확도   | 관세·부가세 자동 계산 |
| **운영 방식**         | 간이 DB 또는 관세청 API 연동 | 자체 DB + 세율테이블 운영      |
| **오류 리스크**        | 낮음 (개인소액면세 범위 많음)   | 높음 (사업자 수입, 오류 시 과태료) |

## *** 어떻게 시스템화하나?
### 개요
#### 1. 상품명 키워드 기반 자동분류
  - 예: “케이스” → 3926.90 / “티셔츠” → 6109.10 / “패딩” → 8109.10
  - 내부 DB에서 키워드별 대표  HS_CODE를 매핑
#### 2. 관세청 API 또는 자체 HS DB 연동
  - UNI-PASS API로 품목검색 자동화
  - 또는 hscode_master 테이블을 주기적으로 업데이트
#### 3. 장점 vs 단점
| 항목         | 장점                 | 단점                    |
| ---------- | ------------------ | --------------------- |
| **속도/안정성** | 즉시 조회 (DB에서 단순 매칭) | 데이터 최신성은 수동 갱신 필요     |
| **운영 효율성** | SKU 등록 자동화, 오류 감소  | 개별 상품 예외는 수동 조정 필요    |
| **유지보수**   | 연 1회 HS개정 반영만 하면 끝 | 관세청 세율 변동은 별도 업데이트 필요 |
#### 검증 로직
  -  HS_CODE 변경 시 자동 알림
  - 신고 오류 가능성이 높은 품목(예: 전자제품, 식품)은 별도 검수 큐 운영

### DB 구조 예시
