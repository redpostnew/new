# 쿠팡 팔레트/밀크런/쉽먼트 개념

## 1. 개요
물류·배송 환경에서 자주 등장하는 **팔레트(Pallet)**, **밀크런(Milk
Run)**, **쉽먼트(Shipment)** 개념을 구조적으로 정리한 문서

## 2. 핵심 개념 비교 표

| 구분                | 정의                                 | 특징                                           | 주 사용처                |
| ----------------- | ---------------------------------- | -------------------------------------------- | -------------------- |
| **팔레트(Pallet)**   | 화물을 팔레트 위에 올려 묶어서 운송·보관하는 방식       | 상·하차 빠름, 파손 감소, 대량·중량 화물 적합                  | 파주↔쿠팡센터, B2B 대량 운송   |
| **밀크런(Milk Run)** | 여러 공급처를 정해진 루트·시간에 따라 반복적으로 수거/배송  | 재고 최소화, 공급 일정 안정, 제조라인 연속성 강화                |   |
| **쉽먼트(Shipment)** | 주문·출하 단위(Shipment No.)로 묶어 관리하는 방식 | 특정 운송방법이 아닌 ‘출하 단위 관리’ 개념, 여러 박스도 한 출하번호로 관리 | 현재 많이 쓰고 있음 |

### 팔레트(Pallet)
- 목적: 대량·중량 화물 묶어서 운송/보관.
- 형태: 나무·플라스틱 판 위에 화물을 쌓고 래핑.
- 장점: 상·하차 빠름, 파손 줄어듦.
- 사용처: 파주 물류센터→쿠팡센터, B2B 대량 운송.

### 밀크런 택배(Milk Run)
- 목적: 여러 공급처를 하루 코스처럼 돌며 정해진 시간에 수거/배송.
- 특징: 정해진 루트·정해진 시간, 반복 운행.
- 장점: 재고 최소화, 공급 일정 안정적.
- 사용처: 자동차/전자 부품 공급망, 제조 기업.

### 쉽먼트 택배(Shipment)
- Shipment = ‘운송 단위/출하’ 의미.
- 국내에서 흔히 쓰는 “쉬프먼트 택배”는 특정 방식이 아니라 출하 단위 기반 배송을 말함.
- 한 주문(=한 출하 단위)을 기준으로 묶어서 보내는 개념.
- 화물이 여러 개여도 하나의 출하번호(Shipment No.)로 관리.

## 3. 흐름 예시
### 3-1. 팔레트 운송 Flow
    파주 물류센터
          ↓
    같은 품목·대량 화물 팔레트 적재
          ↓
    랩핑 후 픽업 대기
          ↓
    트럭 적재 → 쿠팡센터 도착

### 3-2. 밀크런 택배 Flow (부품 공급망 기준)

    A 파주 물류센터
       ↑
    B 파주 물류센터  ←  밀크런 차량(정해진 루트·정해진 스케줄)
       ↓
    C 파주 물류센터
       ↓
    쿠팡센터 도착 

### 3-3. 쉽먼트 관리 Flow

    고객 주문 생성 → Shipment No. 부여
                ↓
    출고 목록 생성
                ↓
    상품 여러 개라도 동일 Shipment No.로 패킹
                ↓
    택배사 스캔 → 발송
                ↓
    고객은 Shipment No. 중심으로 Tracking

## 4. 요약
-   **팔레트**: 대량 화물 묶음 단위
-   **밀크런**: 루트 기반 정기 수거
-   **쉽먼트**: 출하 단위 관리 개념
