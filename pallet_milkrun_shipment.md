# 쿠팡 팔레트/밀크런/쉽먼트 개념

## 1. 개요
물류·배송 환경에서 자주 등장하는 **팔레트(Pallet)**, **밀크런(Milk
Run)**, **쉽먼트(Shipment)** 개념을 구조적으로 정리한 문서

## 2. 핵심 개념 비교 표

| 구분                | 정의                                 | 특징                                           | 주 사용처                |
| ----------------- | ---------------------------------- | -------------------------------------------- | -------------------- |
| **팔레트(Pallet)** BASIC(1pt 당) | 화물을 팔레트 위에 올려 묶어서 운송·보관하는 방식       | 상·하차 빠름, 파손 감소, 대량·중량 화물 적합                  | 파주↔쿠팡센터, B2B 대량 운송   |
| **밀크런(Milk Run)** 밀크런 택배| 여러 공급처를 정해진 루트·시간에 따라 반복적으로 수거/배송  | 재고 최소화, 공급 일정 안정, 제조라인 연속성 강화                |   |
| **쉽먼트(Shipment)** CJ택배| 주문·출하 단위(Shipment No.)로 묶어 관리하는 방식 | 특정 운송방법이 아닌 ‘출하 단위 관리’ 개념, 여러 박스도 한 출하번호로 관리 | 현재 많이 쓰고 있음 |

### 팔레트(Pallet)
- 목적: 대량·중량 화물 묶어서 운송/보관.
- 형태: 나무·플라스틱 판 위에 화물을 쌓고 래핑.
- 장점: 상·하차 빠름, 파손 줄어듦.
- 사용처: 파주 물류센터→쿠팡센터, B2B 대량 운송.

### 밀크런 택배(Milk Run)
- 목적: 여러 공급처를 하루 코스처럼 돌며 정해진 시간에 수거/배송.
- 특징: 정해진 루트·정해진 시간, 반복 운행.
- 장점: 재고 최소화, 공급 일정 안정적.
- 사용처: 자동차/전자 부품 공급망, 제조 기업.

### 쉽먼트 택배(Shipment)
- Shipment = ‘운송 단위/출하’ 의미.
- 국내에서 흔히 쓰는 “쉬프먼트 택배”는 특정 방식이 아니라 출하 단위 기반 배송을 말함.
- 한 주문(=한 출하 단위)을 기준으로 묶어서 보내는 개념.
- 화물이 여러 개여도 하나의 출하번호(Shipment No.)로 관리.

## 3. 흐름 예시
### 3-1. 팔레트 운송 Flow
    파주 물류센터
          ↓
    같은 품목·대량 화물 팔레트 적재
          ↓
    랩핑 후 픽업 대기
          ↓
    트럭 적재 → 쿠팡센터 도착

### 3-2. 밀크런 택배 Flow (부품 공급망 기준)

    A 파주 물류센터
       ↑
    B 파주 물류센터  ←  밀크런 차량(정해진 루트·정해진 스케줄)
       ↓
    C 파주 물류센터
       ↓
    쿠팡센터 도착 

### 3-3. 쉽먼트 관리 Flow

    고객 주문 생성 → Shipment No. 부여
                ↓
    출고 목록 생성
                ↓
    상품 여러 개라도 동일 Shipment No.로 패킹
                ↓
    택배사 스캔 → 발송
                ↓
    고객은 Shipment No. 중심으로 Tracking

## 4. 요약
-   **팔레트**: 대량 화물 묶음 단위
-   **밀크런**: 루트 기반 정기 수거
-   **쉽먼트**: 출하 단위 관리 개념

### 밀크런 택배 vs 쉽먼트 택배 : 차이점
| 항목               | **밀크런 (Milk Run)**                                                                                      | **쉽먼트 (Shipment)**                                                                                      |
| ---------------- | ------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------- |
| **정의 / 방식**      | 쿠팡같은 물류회사가 **셀러(공급자)를 순회해서 직접 수거**하는 운송 방식. 여러 셀러의 물량을 묶어서 한 경로로 차량이 돌면서 픽업함.             | 셀러가 **자기 상품을 쿠팡 창고(센터)로 입고**할 때 사용하는 개념. 운송 수단은 택배, 트럭, 밀크런 등 다양하게 선택 가능함.          |
| **운송 주체 부담**     | 쿠팡이 픽업 루트를 운영하거나 셀러와 공동 부담하는 경우가 있음. COLOpedia에 따르면 “쿠팡이 직접 방문해서 수거” 방식.  | 셀러가 운송비를 부담함이 일반적 (“셀러 부담”)                                          |
| **입고 예측성 / 스케줄** | 픽업 스케줄이 미리 정해져 있어서 **입고 예측이 높음**. COLOpedia 버전 보면 정기 루트.             | 셀러가 운송 예약 + ASN(입고 통지) 등록을 통해 스케줄 조율 가능. COLOpedia 기준 “셀러 자율성 높음”.   |
| **대상 셀러 유형**     | 중대형 셀러, 물량이 많아 정기적으로 픽업이 유리한 경우에 적합. COLOpedia에서 “중대형 셀러 위주”라고 함.    | 규모 제한이 덜하고, 소형 셀러나 변동 주문량이 있는 셀러도 쉽게 사용 가능. COLOpedia에서 “누구나 가능”이라고 설명됨. |
| **장점**           | - 공차율 감소<br>- 운송 비용 절감<br>- 쿠팡이 직접 수거해서 물류 효율성이 높음                              | - 자율성 높음<br>- 다양한 입고 수단 선택 가능 (택배, 트럭, 밀크런 등)<br>- 유연한 입고 스케줄 조정 가능
