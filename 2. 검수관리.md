# 2. 검수 단계 (촬영/부가서비스)

## 1. 검수 프로세스 개요
* **시작**: 검수자가 상품 박스에 부착된 **QR 코드(주문정보)**를 스캔하면, 모바일/태블릿 화면에 해당 주문의 검수 페이지가 자동으로 열림.

---

## 2. 세부 검수 절차

### 2-1. 기본 촬영 / 검수 / 분류
모든 입고 상품에 대해 수행하는 기초 단계입니다.

1.  **사진 촬영**
    * **방식**:
        * **실사 촬영 (PC)**: 대량 작업 시, 카메라로 촬영 후 엑셀 일괄 업로드.
        * **즉시 촬영 (Mobile)**: 검수 화면 내 '사진 첨부' 버튼을 눌러 바로 촬영 및 업로드.
    * **정책**: 구매대행(무료) vs 배송대행(유료) 구분, 최대 촬영 매수 제한 설정.
2.  **기본 검수 (불량 선별)**
    * **상태값 변경**: `대기중` → `검수완료` 또는 `오류입고`
    * **필수 기록 데이터**:
        * **상태**: 정상 / 오염 / 파손
        * **수량**: 일치 / 불일치
        * **사진**: 파일 경로(Server Path) 또는 클라우드 링크 자동 입력
        * **일시**: 시스템 자동 기록 (`YYYY-MM-DD HH:MM`)
    * **데이터 기록 예시**:
        > `250203-B0001-0099` | `CLO1010-0052-B160` | `가죽가방` | `BLACK` | `1` | `정상` | `/img/ORD12345.jpg` | `2025-09-10 14:25`
3.  **검수 결과 처리**
    * **정상**: 부가서비스 검수 단계로 이동 (원산지 확인 등).
    * **오류입고 (파손/오염/수량부족)**:
        * 보관 단계로 이동.
        * 불량 처리 프로세스 진행 (폐기 / 반품 / 보류 중 택 1).

### 2-2. 부가서비스 검수 (상세 체크리스트)
고객이 요청한 유료/무료 부가서비스를 수행합니다.

* **시스템 로직**:
    * **QR-DB 연동**: QR 코드를 스캔하면 DB에 저장된 해당 주문의 '요청된 부가서비스' 목록을 불러옴.
    * **UserForm 제공**: 검수자 화면에 체크박스(`Checkbox`) 형태의 UI 제공.
    * **자동 완료 처리**: 필수 체크항목 완료 후 저장 시, `검수상태 = 완료`로 자동 변경.
    * **증빙 자료**: 분쟁 대비용 사진/영상을 QR 코드 데이터와 매핑하여 저장.
* **부가 검수 운영**:
    * 기본적으로 엑셀 업로드 등을 통해 등록된 '고객 요청 내역'을 기반으로 검수.

## 3. 부가서비스 상세 작업 리스트 (검수내용)

아래 항목은 작업지시서 또는 검수 화면의 체크리스트로 활용됩니다.

| 구분 | 상세 작업 항목 |
| :--- | :--- |
| **1. 라벨/스티커 작업** | • 원산지 스티커 부착 / 원산지 봉제<br>• 브랜드 로고 스티커 부착<br>• 상품 정보 라벨 부착<br>• 택(Tag) 제거 / 라벨 제거 / 미싱<br>• KC 인증 / 연령제한 스티커 부착<br>• 바코드 라벨 부착<br>• 신발 도장 작업 |
| **2. 포장 / 재포장 작업** | • 개별 포장 / 세트 포장<br>• 박스갈이<br>• 포장 보안 (완충재 보강)<br>• 선물 포장 / 쇼핑백 동봉<br>• 박스 제거 / 마대 제거 |
| **3. 검수 / QC 관련** | • 옵션 수량 확인<br>• 옵션(색상, 사이즈 등) 확인<br>• 구성품 누락 검수<br>• 외관 검수 / 정밀 검수<br>• 바코드 정보 검수 및 스캔 |
| **4. 출고 / 물류 관련** | • 팔레트 적재<br>• 택배 송장 출력 및 부착<br>• 쿠팡 납품용 서류 부착 |
| **5. 사진 / 보고서 서비스** | • 입고 사진 촬영<br>• 출고 전 포장 사진 제공<br>• 상품 실사 촬영<br>• 이상 건(불량) 정밀 보고서 작성 |
| **6. 기타 요청형 작업** | • 상품 분리<br>• 쇼핑몰 입점 서류 동봉<br>• 가격표 부착 또는 제거<br>• 수기 입력 작업 |
| **7. 쿠팡 입고 전용** | • **포장 여부**: 혼합 포장 / 단독 포장<br>• **발송 포장 방법**: 박스(택배) / 팔레트<br>• **배송 방식**: 밀크런 / 쉽먼트 |

## 4. 검수 코드 (Inspection Code) 설계

카테고리별로 상이한 검수 포인트 관리를 위해 고유 코드를 부여하여 관리합니다.

* **기본 원칙**: 모든 상품에 '카테고리별 기본 검수항목'은 항상 포함됨.
* **검수코드 정의**: 검수 체크리스트 종류를 구분하는 고유 ID
    * 형식: `CHK_{카테고리}_{버전}`

**[검수코드 적용 예시]**

| 주문번호 | 박스번호 | 상품정보 | **검수코드** | 설명 |
| :--- | :--- | :--- | :--- | :--- |
| 250203-B0001-0099 | BOX1001 | CLO1010-0052-B160 | **CHK_BAG01** | 가방 전용 체크리스트 (지퍼, 가죽상태 등) |
| 250203-B0002-0099 | BOX1002 | TCLO1010-0051-B130 | **CHK_TOY02** | 장난감 전용 체크리스트 (작동여부, 파손 등) |

## 5. 기대 효과

1.  **검수 기록의 디지털화**: 수기 기록을 없애고, 주문별로 데이터가 자동 저장되어 이력 관리가 투명해짐.
2.  **분쟁 해결 효율화**: 고객 클레임 발생 시 QR 코드와 매핑된 사진/영상을 즉시 조회하여 빠른 대응 가능.
3.  **검수 오류 최소화**: 시스템이 자동으로 띄워주는 체크리스트를 따르므로 누락이나 작업 착오 방지.
4.  **자동 리포트 생성**: 고객에게 제공할 검수 결과 리포트가 자동으로 생성되어 업무 시간 단축.
