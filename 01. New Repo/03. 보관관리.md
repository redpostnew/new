## 보관 단계 (창고 → 출고 전)

### 1. 상품 상태 (Status)
* **출고대기**
    * 결제 후 출고를 기다리는 대기 상태
* **오류입고** (입고 검수 중 문제 발생)
    * **오류무시**: 고객이 "그냥 받겠습니다"라고 요청한 상태
    * **출고보류**: 입고는 되었으나 이슈로 인해 보류된 상태 (재고관리와 유사하지만, 정식 재고로 등록하지 않음)
    * **반품관리** (판매자에게 반송)
        * *반품 트래킹 프로세스*
            1. 반품신청
            2. 결제대기 (반품 배송비 등)
            3. 결제완료
            4. 반품완료
* **재고입고** ("중국 센터에 보관해요")
    * **재고신청**: 고객이 직접 보관을 요청함
    * **재고입고**: 물건이 창고에 도착하여 확인됨
    * **재고등록**: 시스템상 관리 가능한 재고로 확정됨

### 2. 랙 정보 (Location Management)
> **Note**: 현재 구체적인 관리 계획은 없으나, 도입 시 로직은 아래와 같음

* **운영 프로세스**
    * 랙(보관 위치)에 **QR 코드** 부착
    * QR 스캔 시 해당 위치의 **보관 상품 목록** 확인 가능
    * **위치 매핑**: `[상품 QR]` + `[위치 QR]` 연속 스캔 → 시스템에 위치 정보 자동 기록
* **기대 효과**
    * 재고 위치 확인 속도 향상 (↑)
    * 상품 분실 및 누락 방지
