## 배송대행 
### 개요
    - 1688 기반으로 상품 직접 결제 기능없음
    - 결제는 고객이 1688 실제 판매자 페이지로 이동해서 진행
    - 고객의 결제 프로세스는 외부(1688)에서 처리
    - 레드포스트는 단순히 [상품찾기]를 통해 장바구니와 결제지원만 하며, 결제와 고객이 직접 진행하고 통관은 레포에서 진행함
    - 해외 사용자는 결제 제한이 많기 때문에 대행업체(소싱대행사)를 통해 결제하는 경우가 일반적
### 결제 방식

- 알리페이(Alipay, 支付宝) 사용이 기본
- 중국 내 은행계좌나 카드필요 필요
- 카드는 해외카드도 가능함
- 결제수단
  - 카카오페이
  - 신용카드
  - 구글플레이
  - 애플페이

### 배송대행 절차
-  슈퍼레이블 참조 / https://super-label.com/
#### 주문/검수
    0. 장바구니: 고객이 검색 후 장바구니 담기
    1. 결제대기: 장바구니에 담고 견적신청한 상태, 주문번호 생성
          1. 상품을 선택하고 1688 [주문서 생성]
          2. [주문서 생성]을 하면 1688 본인 계정에도 주문서가 들어감
          3. 결제전 고객이 판매자와 가격협상 및 소통하여 단가 수정
          4. 신용카드/카카오페이/알리페이/월드퍼스트(1688에 결제)
             - 1688 연동(처음 1회만 필요)
             - 1688 직접결제>알리페이(쯔프바오)결제
             - 1688 직접결제>카드등록 및 카드결제
             - 1688 직접결제>카카오페이 결제
             - 월드퍼스트 결제
                - 월드퍼스트 계정 연동(처음 1회만 필요)
    2. 구매완료
       1. 결제완료하면 구매리스트에 노출
       2. 체크박스로 클릭하여 배송대행 신청서 작성
       3. or 엑셀로 작성후 신청
    3. 접수신청
    4. 입고완료
    5. 재고출고요청: 
       - 창고에 있는재고를 출고요청(창고->출고전 상품임)
       - 고객요청이 있을 때에만 노출함
    6. 검수완료
    7. 출고요청
    8. 포장완료
    9. 결제대기
#### 배송
    1.  결제완료: 무통장입금/신용카드/예치금/알리페이 = 레드포스트에 결제
    2.  출고완료 
    3.  배송완료 

### 배송대행 정산방법 (1+2)

   2. 배송비 (국제배송비와 국내택배비)

배송대행 서비스의 정산 금액을 산출하기 위한 기준 및 공식 정의서입니다.

| 구분 | 항목 | 산출 근거 및 공식 | 데이터 연동 및 비고 |
| :---: | :--- | :--- | :--- |
| **1** | **배송비** | • **산출식**: `국제 배송비 + 국내 택배비`<br>해운특송과 항공건만 국내택배비 포함<br> LCL/FCL은 국내택배비용 불포함<br><br>• **적용 중량 기준**<br>실중량(Actual Weight)과 부피무게(Volume Weight) 중 **더 큰 값(Max)**을 적용<br><br>• **부피무게 공식** | • **요율표 연동**<br>관리자가 입력한 배송비 요율표에 따라 금액이 책정됩니다.<br><br>• **패킹리스트 연동**<br>패킹리스트의 측정 데이터(CBM, KG)와 연동하여 1차 자동 계산됩니다. |

    12. 출고완료

### 결제 방법
- 무통장 입금
- 신용카드/체크카드 - 나이스페이먼츠
  - 신용카드 수수료 적용
- 실시간 이체
- 가상계좌
- 예치금
  - 원화 입금계좌
  - 외화 입금계좌(자동 환율 계산 및 결제 수수료 표시 기능)

### 특징
- 결제 이후 단계(수입신고·배송대행)까지 시스템에서 관리 가능
- 구매-결제-배송 과정을 통합 관리 가능
- 1688 결제 경험이 없는 사용자도 원화로 쉽게 결제할 수 있게 설계함
- 관리자는 실시간 마진 계산, 정산 리포트

### 구매대행 정산방법 (1+2+3)
0.  주문정보
1.  상품/물류 비용
    - 개별상품으로 하지 않고 상품 총비용으로 함
    - 정산구조    
      - 상품값 = 상품가격 + 중국내 배송비(1688 판매자가 지정)
      - 수수료 = 상품총비용* 수수료율(관리자 넣음) 
2. 부가 서비스 
    부가서비스 처음 세팅 = 관리자
    주문번호별 부가서비스 1차 자동계산
    중국 관리자가 내용 최종 수정/확정
    부가서비스는 상품별로 정산함
3. 배송비 (국제배송비와 국내택배비)
    물류
        가로(cm) × 세로(cm) × 높이(cm) / 6,000 = 부피 중량(kg)
        실제 중량과 부피 중량 중 더 큰 값을 청구 중량으로 사용
        CBM은 LCL, FCL만 적용함

### 구매대행 정산 산출 근거

| 항목 | 산출 근거 및 공식 | 상세 / 비고 |
| :--- | :--- | :--- |
| **1. 상품 총비용** | **상품 가격 + 중국 내 배송비**<br>(개별 상품이 아닌 전체 상품 총액 기준) | • 주문 정보에서 총비용 산출 |
| **2. 대행 수수료** | **상품 총비용 × 수수료율**<br>(관리자 설정값 적용) | • 관리자 수수료율 설정 메뉴에 따름 |
| **3. 배송비** | **국제 배송비 + 국내 택배비**<br><br>👉 **중량 적용:** `MAX(실중량, 부피무게)`<br>👉 **부피무게:**  | • 패킹리스트 연동 1차 자동 계산<br>• 배송비 요율표 기준 관리자 입력 |
| **4. 부가 서비스 비용** | **주문번호별 1차 자동 계산**<br>$\rightarrow$ **관리자 최종 확정** | • 상품별 정산 |

## 수입대행
### 견적방식
- 공급가 견적 = 모든비용을 한국도착가격으로 견적&정산
    - 결제1회(선결제 or 한국출고전 결제), 모든 비용을 계산해서 한번에 청구하여 받음
    - 모든비용=상품가+중국내배송비+국제운송비+부가서비스비용+통관비+관세+부가세+국내내륙배송비+마진

    2.  결제대기

    4.  구매완료 
    5.  입고완료

    7.  검수완료
    8.  출고요청
    9.  포장완료
    10.  출고완료

### 수입대행 정산방법

## 공식
### 구매대행

* **1차결제**
    * 총 결제 금액 = 상품 총 금액(KRW) = `상품 총 금액(KRW) + (상품단가(CNY) x 환율 + 중국 내 배송비(CNY) x 환율) x 구매대행 수수료율(%공통설정/개별설정가능)`
* **2차결제**
    * 총 결제 금액 = `국제 운송비(KRW) + 부가서비스비용(KRW)`

### 배송대행

* **1차결제**
    * 고객이 1688에 직접 결제한 금액 (정산에는 미포함)
* **2차결제**
    * 총 결제 금액 = `국제 운송비(KRW) + 부가서비스비용(KRW)`

### 수입대행

* **1차결제**
    * 상품 총 금액(KRW) = `상품 총 금액(KRW) + (상품단가(CNY) x 환율 + 중국 내 배송비(CNY) x 환율) x 수입 수수료율(%공통설정/개별설정가능)`
    * 공급가 = `상품 총 금액(KRW) + 국제 운송비(KRW) + 부가서비스비용(KRW) + 통관비(상품 총 관세(KRW) + 상품 총 관세(KRW) * 통관 수수료(%공통설정))(KRW) + 한국 내 배송비(KRW)`
    * 마진 = `공급가 x 마진율(25% 공통설정)`
    * 부가세 = `(공급가 + 마진) * 10%(공통설정)`
    * 총 결제 금액 = `공급가 + 마진 + 부가세`
